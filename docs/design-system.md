# 约好猫 Design System

Version 2.0

本规范以当前 Figma 成稿为准，用于后续网页实现、Figma 设计稿维护和组件迭代。原则是少变量、真应用、强一致：只保留当前产品真实用到的颜色、字号、圆角和阴影，不为了“丰富”创建多余层级。

## 1. 基础原则

- 移动端优先：核心画布为 `390 x 844`。
- 视觉优先级：画面完整度、和谐度、无重叠高于功能堆叠。
- 信息克制：每个页面只保留当前任务需要的信息，不和上一级重复。
- 组件统一：同类按钮、卡片、标签、返回按钮、底部操作区必须保持一致。
- 图片与图标：优先复用 Figma 中已经确认的图片和图标；缺失时使用与猫相关的简洁插画或黄色占位。

## 2. Color Tokens

色值来自当前 Figma 色板截图取样。遇到不确定颜色时，优先从这组颜色中选择，不新增临时色。

| Token | 色值 | 用途 |
| --- | --- | --- |
| `color.page` | `#fffef9` | 页面背景色 |
| `color.card` | `#ffffff` | 卡片、输入框、底部栏背景 |
| `color.text.primary` | `#3a3a3a` | 主文本、大标题、重要图标 |
| `color.text.secondary` | `#736d66` | 正文、说明文字、次级导航 |
| `color.text.placeholder` | `#a8a095` | 搜索框文字、输入提示 |
| `color.primary` | `#fdca18` | 主题黄、主按钮、顶部区、选中色、价格 icon |
| `color.primary.strong` | `#8a6400` | 强调文本、选中 icon、价格相关强调 |
| `color.accent.pale` | `#fff5c7` | 浅黄标签、价格块、部分 icon 填充 |
| `color.accent.pink` | `#fde1e1` | 猫爪点缀、温柔气质辅助色 |
| `color.tag.neutral` | `#f4f4f1` | 猫卡片中的小标签底色 |
| `color.border` | `#eee9df` | 卡片描边、分割线 |
| `color.success` | `#68b984` | 预约成功 icon |

使用规则：

- 主按钮使用 `color.primary` 背景，文字使用 `color.text.primary`。
- 选中 icon 和强调文字使用 `color.primary.strong`。
- 大段说明和普通正文使用 `color.text.secondary`，不要随意加深。
- 搜索框文案使用 `color.text.placeholder`。
- 卡片描边只用 `color.border`，保持轻，不做明显深边框。
- 不再使用蓝色作为主色、选中色或交互色。

## 3. Typography

字体族：`HarmonyOS Sans SC`, `PingFang SC`, `Microsoft YaHei`, `system-ui`, `sans-serif`。

当前只允许以下字号：

| Token | 字号 / 建议行高 | 用途 |
| --- | --- | --- |
| `type.title` | `22px / 30px` | 详情页猫名、成功页主文案、重要标题 |
| `type.navTitle` | `20px / 28px` | 二级页顶部标题、模块标题 |
| `type.body` | `16px / 24px` | 按钮文字、正文、主要说明 |
| `type.meta` | `14px / 20px` | 标签、辅助信息、日期时间说明 |
| `type.caption` | `12px / 16px` | 价格副文案、底部导航、极小提示 |

字重规则：

- 重要标题使用中高字重，保持清楚但不要笨重。
- 正文和说明大幅降级，使用常规字重。
- 标签文字不加粗，避免小组件显得拥挤。
- 不使用 24、28、32 等旧大字号；页面标题不得压过内容主体。

## 4. Radius

圆角只保留两种：

| Token | 值 | 用途 |
| --- | --- | --- |
| `radius.large` | `24px` | 大卡片、详情内容面板、底部弹层、首页功能按钮 |
| `radius.medium` | `16px` | 小按钮、标签、返回按钮、日期/时间按钮、内部卡片 |

规则：

- 不再新增 18、20、22、28、30 等中间圆角。
- 外层大容器用 `24px`，内部组件统一用 `16px`。
- 图片与白色信息区衔接处也按 `24px` 和 `16px` 的关系处理，不能出现尖角或错位。

## 5. Shadow

阴影只用于让白色卡片从页面背景中轻轻浮起，不能明显“飘起来”。

| Token | 建议值 | 用途 |
| --- | --- | --- |
| `shadow.soft` | `0 8px 24px rgb(58 58 58 / 5%)` | 标准卡片 |
| `shadow.float` | `0 10px 28px rgb(58 58 58 / 6%)` | 底部弹层、较高层级卡片 |

规则：

- 不使用重阴影。
- 黄色按钮一般不需要彩色阴影。
- 描边和留白优先，阴影只是辅助。

## 6. Spacing And Grid

- 页面左右主边距：`20px`。
- 卡片内边距：`20px` 或 `16px`，根据卡片大小选择。
- 组件间距优先使用：`8 / 12 / 16 / 20 / 24 / 32`。
- 底部导航高度：约 `65px`。
- 底部固定操作区高度：以 `75px` 左右为基准，按钮高度 `48px`。
- 返回按钮尺寸：`40 x 40`。

## 7. Components

### Primary Button

- 高度：`48px`
- 圆角：`16px`
- 背景：`color.primary`
- 文字：`16px`，主文本色
- 用途：确认预约、预约陪伴、查看我的预约、猫咪卡片 CTA

### Secondary Button

- 高度：`48px`
- 圆角：`16px`
- 背景：`color.card`
- 描边：`color.border`
- 文字：`16px`，主文本或次级文本

### Tag

- 圆角：`16px`
- 字号：`14px`
- 浅黄标签：`color.accent.pale` + `color.primary.strong`
- 中性标签：`color.tag.neutral` + `color.text.secondary`

### Cat Card

- 外层圆角：`24px`
- 描边：`color.border`
- 阴影：`shadow.soft`
- 图片区域与信息区域必须自然过渡，白色信息区顶部保留圆角。
- CTA 文案统一为：`立即预约`

### Detail Page

- 顶部大图保留沉浸感。
- 内容面板从图片下方以大圆角承接。
- 信息结构优先级：猫名、年龄性别、状态标签、关键属性、适合的人、互动建议。
- 底部固定操作：收藏按钮 + 预约陪伴按钮。

### Time Picker Sheet

- 从猫咪详情页点击预约陪伴后出现。
- 作为底部上滑卡片，不再跳转独立预约页和确认页。
- 结构：猫咪摘要、日期、时间、合计费用、确认预约。
- 选择完成后直接进入预约成功页。

### Bottom Navigation

- 四个固定入口：首页、陪伴、预约、我的。
- 当前页使用 `color.primary.strong`。
- 未选中状态使用 `color.text.secondary`，降低视觉权重。

## 8. Interaction Flow

当前预约流程：

1. 首页或陪伴页进入猫咪详情。
2. 在猫咪详情页点击 `预约陪伴`。
3. 底部滑入选择时间卡片。
4. 选择日期和时间后显示合计费用。
5. 点击 `确认预约`。
6. 直接进入预约成功页。

旧流程中的独立「预约陪伴页」和「确认预约页」不再作为真实用户路径保留。

## 9. Asset Rules

- Figma 中如果图片是独立图片填充，可以导出并复用为网页图片。
- Figma 中如果 icon 是 vector / SVG 节点，可以导出为 SVG 并复用。
- 如果图片或 icon 已经被合成在一张截图里，只能作为参考，不能直接拆出可编辑层。
- 网页实现时优先保留 Figma 中确认过的图形语义和比例，不重新发明图标风格。

## 10. 禁止项

- 禁止新增未确认颜色变量。
- 禁止使用旧蓝色体系。
- 禁止使用 22/20/16/14/12 以外的字号，除非先重新确认规范。
- 禁止使用 24/16 以外的圆角。
- 禁止重阴影。
- 禁止按钮无路径、路径不闭环。
- 禁止页面内容与上一级重复堆叠。
